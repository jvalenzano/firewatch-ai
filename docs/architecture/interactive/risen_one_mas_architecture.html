üèóÔ∏è ADK Multi-Agent Architecture Notes
Agent Orchestration: The root agent receives user queries, determines which sub-agents to involve, and coordinates their responses into a unified answer.
Sub-agent Specialization: Each sub-agent focuses on a specific domain (weather, fire risk, ML predictions) and can maintain its own conversation context and reasoning.
Tool Integration: Both root and sub-agents can call tools to perform specific functions like data retrieval, calculations, and visualization generation.
Scalable Design: New sub-agents and tools can be added without modifying existing components, following ADK's modular architecture principles.
Session Management: Each agent maintains its own session state while the root agent coordinates overall conversation flow and context sharing.
